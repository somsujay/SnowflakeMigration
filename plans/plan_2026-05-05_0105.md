# Teradata to Snowflake Migration - Medallion Architecture

## Context

**Source**: `Teradata_Scripts/teradata_dwh_etl.sql` (608 lines)
- 3 staging tables, 4 dimension tables, 2 fact tables, 7 stored procedures
- Teradata-specific: `PRIMARY INDEX`, `REPLACE PROCEDURE`, `UPDATE...FROM`, `EXTRACT(DOW)`, `DATE()` cast

**Target Architecture**: Medallion (Bronze / Silver / Gold) in one database with 3 schemas.

## Medallion Layer Mapping

| Layer | Schema | Contents |
|-------|--------|----------|
| **Bronze** | `BRONZE` | Raw staging tables (T_Customer, T_Account, T_Transaction) - 1:1 from source with metadata columns |
| **Silver** | `SILVER` | Cleaned dimensions (DimCustomer SCD-2, DimAccount SCD-1, DimTransactionType, DimDate) |
| **Gold** | `GOLD` | Business-ready facts (FactDailyTransaction, FactDailyAgg) |

## Implementation Steps

### 1. Create `01_setup_schemas.sql`
- `CREATE SCHEMA IF NOT EXISTS BRONZE;`
- `CREATE SCHEMA IF NOT EXISTS SILVER;`
- `CREATE SCHEMA IF NOT EXISTS GOLD;`

### 2. Create `02_bronze_tables.sql`
- Convert T_Customer, T_Account, T_Transaction to `BRONZE.T_Customer` etc.
- Remove `PRIMARY INDEX`
- Add metadata columns: `_LOADED_AT TIMESTAMP_NTZ DEFAULT CURRENT_TIMESTAMP()`
- Use `CREATE OR REPLACE TABLE`

### 3. Create `03_silver_tables.sql`
- Convert DimCustomer, DimAccount, DimTransactionType, DimDate to `SILVER.*`
- Remove `PRIMARY INDEX`
- Add `CLUSTER BY (Date_Key)` to DimDate

### 4. Create `04_gold_tables.sql`
- Convert FactDailyTransaction, FactDailyAgg to `GOLD.*`
- Remove `PRIMARY INDEX`
- Add `CLUSTER BY (Date_Key)` to both fact tables

### 5. Create `05_silver_procedures.sql`
Convert 5 procedures with these transformations:
- `REPLACE PROCEDURE` -> `CREATE OR REPLACE PROCEDURE`
- Add `RETURNS STRING LANGUAGE SQL AS`
- Rewrite `UPDATE...FROM` join syntax
- Use `:param` colon prefix for variables in SQL statements
- `EXTRACT(DOW FROM x)` -> `DAYOFWEEK(x)`
- `DATE(x)` -> `x::DATE`
- DimDate: Replace WHILE loop with `TABLE(GENERATOR(ROWCOUNT => DATEDIFF(DAY, ...)))` approach
- Qualify all table references with schema (e.g., `BRONZE.T_Customer`, `SILVER.DimCustomer`)

### 6. Create `06_gold_procedures.sql`
Convert 2 fact-load procedures:
- Same syntax transformations as Step 5
- Cross-schema references: read from `BRONZE.*`, write to `GOLD.*`
- `DATE(t.Transaction_Date)` -> `t.Transaction_Date::DATE`

### 7. Create `07_orchestration.sql`
- Convert `Daily_ETL_Run` procedure
- Add optional `CREATE TASK` example (commented out, for reference)

### 8. Create `08_seed_data.sql`
- Stage creation + COPY INTO for sample CSVs into BRONZE tables

## Key Conversion Rules Applied

| # | Teradata | Snowflake |
|---|----------|-----------|
| 1 | `PRIMARY INDEX (col)` | Removed |
| 2 | `REPLACE PROCEDURE` | `CREATE OR REPLACE PROCEDURE ... RETURNS STRING LANGUAGE SQL AS` |
| 3 | `UPDATE D FROM S WHERE ... SET ...` | `UPDATE D SET ... FROM S WHERE ...` |
| 4 | `EXTRACT(DOW FROM x)` | `DAYOFWEEK(x)` |
| 5 | `DATE(x)` | `x::DATE` |
| 6 | `INTERVAL '1' DAY` | `DATEADD(DAY, 1, date)` |
| 7 | Parameters in SQL | `:param` colon prefix |
| 8 | `DECLARE v; SET v = x` | `LET v := x` |
| 9 | `DATE '9999-12-31'` | `'9999-12-31'::DATE` |
